# 프로젝트 요약서 및 작업 지시서: TA WatchDog 모바일 앱 전환

## 1. 프로젝트 개요

* **목표:** 기존 텔레그램 봇(`Butler.py`)으로 동작하던 개인 재무 관리(Total Account Tracker) 시스템을 **FastAPI 기반의 백엔드 서버**와 **생체인증이 적용된 안드로이드 네이티브(또는 Flutter) 앱**으로 마이그레이션한다.
* **주요 기능:**
  1. 전체 계좌 및 변동 내역 요약 조회 (일간/월간 대시보드 및 Matplotlib 차트)
  2. 수동 입력 데이터(Manual Inputs) 및 비율(ss.ratio) 관리
  3. OpenAI Function Calling 기반의 자연어 금융 질의응답 (ASK 기능)

## 2. 시스템 아키텍처 및 기술 스택

* **인프라 환경:** Ubuntu 기반의 Docker 컨테이너 환경으로 백엔드 배포
* **데이터베이스:** 기존 운영 중인 PostgreSQL 활용 (CloudBeaver/pgAdmin 등을 통해 관리하기 용이하도록 표준 SQL 적용)
* **백엔드 (API 서버):** Python 3.10+, FastAPI, SQLAlchemy, Pydantic, PyJWT
* **프론트엔드 (안드로이드 앱):** Flutter (크로스 플랫폼) 또는 Kotlin
* **보안/인증:** JWT (JSON Web Token) + 안드로이드 Keystore 기반 생체 인식(Biometric Auth)

## 3. 핵심 구현 요구사항

### Phase 1: 데이터베이스 마이그레이션 (Pickle -> PostgreSQL)

기존 로컬 파일 시스템에 의존하던 `pickle` 데이터를 DB 테이블로 이관한다.

* `manual.pickle`을 대체할 `manual_inputs` 테이블 생성 (`id`, `key_name`, `value`, `updated_at` 컬럼 포함).
* `ss.ratio` (리스트/딕셔너리 형태)를 관리할 설정 테이블 생성.
* 기존 `Butler.py`의 `read_pickle`, `write_pickle` 로직을 SQLAlchemy를 이용한 DB CRUD 로직으로 전면 교체.

### Phase 2: 백엔드 API (FastAPI) 개발

기존 텔레그램 봇의 핸들러를 RESTful API 엔드포인트로 변환한다.

* **[POST] /auth/login:** 초기 1회 로그인을 위한 마스터 비밀번호 검증 및 JWT 발급.
* **[GET] /api/dashboard/summary:** `summary_data()`, `summary_month()`, `diff_detail()` 결과를 JSON 포맷으로 반환.
* **[GET] /api/dashboard/chart:** 생성된 Matplotlib 이미지(`summary.png`)를 바이트 스트림이나 정적 파일 URL로 반환.
* **[GET, POST, DELETE] /api/manual_inputs:** 수동 입력 데이터 조회, 추가, 수정, 삭제 처리 (`manualAdd`, `manualEdit`, `manualDel` 대체).
* **[POST] /api/chat/ask:** `call_llm_with_tools` 함수와 연동하여 사용자의 자연어 질문을 처리하고 답변을 반환.
* **보안 미들웨어:** `/api/*` 경로의 모든 요청은 헤더에 유효한 `Authorization: Bearer <JWT>`가 포함되어 있는지 검증.

### Phase 3: 안드로이드 앱 보안 및 인증 흐름 구현

* **초기 설정:** 앱 최초 실행 시 로그인 화면 노출 -> 마스터 비밀번호 입력 후 백엔드 `/auth/login` 호출 -> 반환받은 JWT를 기기의 안전한 영역(Android Keystore 기반의 EncryptedSharedPreferences 또는 Flutter Secure Storage)에 저장.
* **일상적 실행:** 이후 앱 실행 시 **지문/Face ID 생체 인식 프롬프트** 호출 -> 인증 성공 시 보안 저장소에서 JWT를 꺼내어 백엔드 API 호출 시 헤더에 주입.

### Phase 4: 안드로이드 UI 개발

* **Dashboard Tab:** 상단에 요약 텍스트 배치, 하단에 API로 받아온 차트 이미지 렌더링.
* **Manual Inputs Tab:** 현재 입력된 수치들의 리스트 뷰 표시. FAB(Floating Action Button)을 통한 항목 추가, 스와이프하여 삭제, 탭하여 값 수정하는 직관적인 모달 UI.
* **ASK Chat Tab:** 일반적인 메신저 형태의 UI 구성. 사용자가 텍스트를 입력하면 로딩 인디케이터("🤔 Thinking...") 표시 후 백엔드의 LLM 응답을 말풍선으로 렌더링.

## 4. Antigravity를 위한 작업 지시 (Action Items)

이 지시서를 읽은 후, 가장 먼저 아래 항목들을 순서대로 제시해 주십시오.

1. `manual_inputs`와 `ss.ratio`를 저장할 **PostgreSQL 테이블 생성용 DDL (SQL) 스크립트** 작성.
2. 위 명세를 바탕으로 한 **FastAPI 디렉토리 구조 및 초기 `main.py` 뼈대 코드** 작성 (JWT 발급 및 검증 로직 포함).
